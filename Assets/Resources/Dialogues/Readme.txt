每一个json描述了对应关卡开始时的对话。

对于键 dialogs，它的值为一个列表，列表元素类型为字符串列表，可以认为其描述了一个有向无环图(拓扑图)。
具体来说：
对于键 dialogs 的值中的每一项列表，列表的第一项描述了本条对话的类型。
对话的索引被定义为键 dialogs 的值中列表的索引。
对话类型：
0 - 普通对话(对话框)。
1 - 冥想(背景变暗，中间显示文字)。
2 - 选择(选项)。
3 - 播放 CG。
如果第一项为 "0"：
    第二项描述了说话的角色(Speaker)。
    第三项描述了说话的内容(Content)。
    第四项描述了本条对话结束后，跳转到的另一条对话的索引。
    例如，["0", "Adolf Hitler", "Sieg Heil!", "1"] 表示 Adolf Hitler 说 "Sieg Heil!"，单击回车键(Enter)后跳转到索引为 1 的对话。
如果第一项为 "1":
    第二项描述了冥想的内容(Content)。
    第三项描述了本条对话结束后，跳转到的另一条对话的索引。
    例如，["1", "[你充满了决心。]", "2"] 表示冥想内容 "[你充满了决心。]"，单击回车键(Enter)后跳转到索引为 2 的对话。
如果第一项为 "2":
    第二项描述了选项的个数 count 的字符串。
    此后的 2 * count 项，每两项分别描述了选项文本的内容(Content)和单击该选项后，跳转到另一条对话的索引。
    例如，["2", "3", "[谁问你了？]", "1", "[你的帽子怎么尖尖的？]", "2", "[有点意思。]", "3"] 表示
        3 个选项，点击 [谁问你了？] 跳转到对话 1，点击 [你的帽子怎么尖尖的？] 跳转到对话 2，点击 [有点意思。] 跳转到对话 3。
如果第一项为 "3":
    第二项描述了 CG 视频文件在Resources/下的相对路径。
    第三项描述了 CG 视频播放完成后，跳转到的另一条对话的索引。
